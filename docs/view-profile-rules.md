# View Profile Rules — Reference

Rules for generating per-dataset JSON profiles that drive the 4-view UI:
**Timeline** | **Snapshot** (period browser) | **Table** | **Custom Charts** (later)

Generated by `generate_view_profiles.py`. Output: `data/view-profiles/{CODE}.json`

---

## 1. Dimension Classification

Each dimension is classified by type (from `dimension_options_parsed.dim_type` majority vote) and by these derived properties:

| Property | Logic |
|---|---|
| **type** | `time`, `geo`, `gender`, `age`, `residence`, `unit`, `indicator` |
| **is_singleton** | `option_count == 1` OR (`option_count == 2` AND has "Total" option) → 1 real value → **excluded from charts/filters** |
| **cardinality** | `low` (2-5), `medium` (6-25), `high` (26-100), `very_high` (>100) |
| **is_caen** | `dim_label` matches CAEN/activit patterns |
| **has_hierarchy** | >50% of options have `parent_id != 0` |
| **has_total** | Any option label matches `Total`, `TOTAL`, `Total ` |
| **multi_unit** | Unit dim with `option_count > 1` → **mandatory top-level selector** |

### Special dimension rules

- **Singleton dims**: Don't appear in any chart axis, series, or filter. Listed in `dimensions.singleton_dims` for reference.
- **Multi-unit**: Forces a UM selector scoped to both views (`scope: "page"`). Never aggregate across units.
- **CAEN dims** (high-cardinality industry codes): Always use `typeahead_select` control, max 8 simultaneous series.
- **Hierarchical dims**: Use `tree_select` control. Only selected level used in queries.

## 2. Timeline View

**Available when**: `has_time AND time_points >= 3`

Shows dynamics over time. X-axis is always the time dimension.

### Chart selection (priority order)

#### 2.1 Line chart (always primary when timeline available)

- **x_axis**: time dimension
- **series**: Pick best analysis dim by preference order:
  1. `gender` (if present, cardinality low)
  2. `residence` (if present)
  3. Smallest `indicator` dim with `option_count <= 8`
  4. `age` (if `option_count <= 8`)
  5. No suitable dim → single line (no series)
- **Toggles**:
  - `time_points < 5` → add `bar_vertical` toggle (bars better for short series)
  - No negatives AND series dim with 2-6 options → add `area_stacked` toggle
- **max_series**: 8 (for high-cardinality series dims)

#### 2.2 Additional line variants

When dataset has **2+ non-singleton analysis dims**: generate variant chart entries where each unused analysis dim becomes the series, and the rest become filters.

Example for TUR105G (has `tipuri_de_structuri` + `tipuri_de_turisti`):
- Chart 0: series = `tipuri_de_structuri`, filter = `tipuri_de_turisti` + `geo`
- Chart 1: series = `tipuri_de_turisti`, filter = `tipuri_de_structuri` + `geo`

### Timeline controls

| Condition | Control | Default | Scope |
|---|---|---|---|
| `has_geo` | `single_select` on geo | "Total" / national level | `view` |
| Indicator with cardinality > 25 | `typeahead_select`, max 8 | first 5 options | `view` |
| `is_caen` | `typeahead_select`, max 8 | — | `view` |
| `has_hierarchy` | `tree_select` | root level | `view` |
| `multi_unit` | `single_select` on unit | first unit | `page` |
| Low-cardinality analysis dim as filter | `pill_group` | "Total" if has_total, else first | `view` |
| Medium-cardinality (6-25) | `multi_select` dropdown | first 5 | `view` |

## 3. Snapshot View

**Available**: Always (even without time dimension — becomes the only view)

Shows data for a single time period with a period browser.

### Period browser

- When `time_points >= 2`: show `< prev | current | next >` navigation
- **Matches source granularity**: monthly → browse months, annual → years, quarterly → quarters
- Default: latest available period
- When no time dimension: no period browser, snapshot is static

### Chart selection (priority order)

#### 3.1 Choropleth (when `has_geo AND geo_count >= 5`)

- **x_axis**: geo dimension (map position)
- All analysis dims → single-select filters (choropleth shows one scalar per geo unit)
- Score boost if `geo_count >= 30` (near-complete county coverage)
- **Not eligible** when geo is locality-level only (no GeoJSON)

#### 3.2 Bar chart (almost always available)

- **With geo**: `horizontal_bar`, geo on y-axis as ranking
  - Optional series from low-cardinality dim (gender, residence)
  - Toggle: `stacked_bar` / `grouped_bar` when series dim has 2-4 options
- **Without geo**: `bar_vertical` or `grouped_bar`
  - x_axis: largest analysis dim (≤20 options)
  - series: second analysis dim (if exists, ≤6 options)
  - Toggle: `stacked_bar` / `grouped_bar` when series has 2-4 options

#### 3.3 Population pyramid (when `has_age AND has_gender AND gender_count in [2, 3]`)

- **Primary over grouped_bar** when both age and gender present
- x_axis: age, series: gender
- Excludes "Total" from gender automatically

#### 3.4 Bubble/matrix (when 2+ analysis dims with cardinality >= 3 each)

- Cross-tabulation of two dimensions
- x_axis: first qualifying dim, series: second
- If >2 qualifying dims → `dimension_pair_toggle` control for choosing which pair

#### 3.5 Heatmap (when 2+ analysis dims, at least one >= 10 options, `fill_rate > 0.3`)

- Dense cross-tabulation view
- x_axis: highest-cardinality dim, y_axis: second-highest

### Snapshot controls

Same rules as Timeline controls, plus:

| Condition | Control | Default | Scope |
|---|---|---|---|
| `time_points >= 2` | `period_browser` | latest period | `view` |
| Choropleth active | Analysis dims become `single_select` | "Total" if exists | `chart:choropleth` |

## 4. Table View

**Available**: Always.

No chart-specific rules. All dimensions become filter controls. Data displayed in paginated table with sort.

---

## 5. Control Scoping

Controls have scope:
- `"page"` — affects all views (e.g., UM selector)
- `"view"` — affects all charts within timeline or snapshot
- `"chart:{type}"` — affects only a specific chart (e.g., choropleth needs single-select where bar allows multi-select)

### Control type by cardinality

| Cardinality | Non-series (filter) | Series |
|---|---|---|
| low (2-5) | `pill_group` | shown directly as series |
| medium (6-25) | `multi_select` dropdown | `multi_select`, max 8 |
| high (26-100) | `single_select` dropdown | `typeahead_select`, max 8 |
| very_high (>100) | `typeahead_select` | `typeahead_select`, max 8 |

### Toggle groups

`stacked_bar` / `grouped_bar` / `line` are variants of the same visual, rendered as one chart container with a toggle button — not separate chart containers.

---

## 6. Warnings

| Condition | Type | Severity |
|---|---|---|
| `fill_rate < 0.25` | `sparse_data` | `warning` |
| `fill_rate < 0.10` | `very_sparse` | `warning` |
| `time_points < 3` | `short_series` | `info` |
| Non-geo dim with `option_count > 100` | `high_cardinality` | `info` |
| `multi_unit` | `multi_unit` | `info` |

---

## 7. Edge Cases

- **No time dimension**: Snapshot only, no period browser, timeline unavailable
- **Regional-only geo** (no counties): No choropleth (too few units), use grouped bar instead
- **Locality-level geo** (thousands of values): No choropleth (no GeoJSON), use typeahead filter
- **Composite dims** (e.g., TCJ0331 with sex+age+edu+region mixed): Treat as flat indicator, no semantic grouping
- **CAEN hierarchy** (e.g., INT109A with parent-child columns): Detect consecutive CAEN dims, collapse into tree_select
- **Datasets with only unit + time** (2 dims): Single line chart, no filters needed
